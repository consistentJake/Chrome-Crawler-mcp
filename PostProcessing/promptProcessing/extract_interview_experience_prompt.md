# 中文面试经验信息提取Prompt

## 任务目标
从JSON格式的论坛帖子中提取有价值的中文面试经验信息，包括面试题目、准备建议和有用的回复讨论。

## 输入格式
JSON文件包含多个帖子，每个帖子有：
- url: 帖子链接
- thread_title: 帖子标题
- replies: 回复内容数组

## 提取要求

### 1. 面试题目信息
提取以下类型的面试题目：
- 算法题/编程题（包括具体题目描述）
- 系统设计题（包括系统要求和功能）
- 机器学习/AI相关题目
- 行为面试/项目经历问题
- 数学/概率题

### 2. 有用的准备建议
提取：
- 题目准备建议
- 面试技巧分享
- 考察重点提示
- 时间分配建议

### 3. 额外有价值信息
提取：
- 面试流程描述
- 面试时长（60分钟/75分钟等）
- 面试团队/职位信息
- 题目类型分类（新题/旧题）
- 经验总结/教训

### 4. 输出格式
对于每个提取的信息项，输出：
```json
{
  "type": "面试题目|准备建议|有用信息",
  "title": "简洁的标题",
  "content": "详细内容",
  "url": "来源链接",
  "timestamp": "如果有时间信息",
  "category": "算法|系统设计|ML|行为面试|其他",
  "notes": "额外备注"
}
```

## 处理规则

1. **中文优先**：主要提取中文内容，英文内容如果重要也可提取
2. **去重**：相同或相似的题目只保留最完整的一个
3. **完整性**：尽量保留题目的完整描述和要求
4. **实用性**：优先提取对后续面试者有实际帮助的信息
5. **分类清晰**：明确标注每类信息的类型和类别

## 特殊处理

- 对于"求分享"、"同问"等无实质内容的回复可以忽略
- 对于明显过时的信息（如多年前的面经）可以标注
- 对于需要积分才能查看的内容，如果部分信息可见也要提取
- 对于交换面经的讨论，提取其中透露的题目信息

## 示例输出

```json
{
  "type": "面试题目",
  "title": "OpenAI怪兽对战题",
  "content": "设计并实现一个核心战斗逻辑，模拟两队怪兽之间的回合制战斗...",
  "url": "https://www.1point3acres.com/bbs/thread-1156671-1-1.html",
  "category": "算法",
  "notes": "60分钟coding题，需要良好的OOD设计"
}
```

请基于以上要求，从提供的JSON文件中提取所有有价值的面试经验信息。
